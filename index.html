Three Paths to Your Virtual Machine

  1. AWS Console - Visual, beginner-friendly
  2. Browser SSH (EC2 Instance Connect) - No local tools needed
  3. AWS CLI - Automation and scripting

We'll walk through all three methods so you understand the full spectrum of options.

Method 1: Console Launch

Key Decisions:

  • AMI: Ubuntu 22.04 LTS (Free Tier eligible)
  • Instance Type: t2.micro (1 vCPU, 1 GB RAM)
  • Key Pair: Create and download .pem file
  • Security Group: Allow SSH from My IP only

Security First: Never expose SSH to 0.0.0.0/0 in production!

Easiest Connection: EC2 Instance Connect

No terminal? No problem!

  1. Select your instance in the EC2 dashboard
  2. Click "Connect"
  3. Choose "EC2 Instance Connect"
  4. Click "Connect" again

A terminal opens directly in your browser!

Model Context Protocol (MCP)

The missing link between AI and your infrastructure

  • Open standard for LLM-tool integration
  • AI agents discover available tools
  • AI agents understand tool capabilities
  • AI agents execute tools securely

Think of it as an API gateway designed specifically for AI agents.

Serverless MCP on AWS

Lambda + API Gateway = Cost-Effective AI Bridge

  • AWS Lambda: Runs your tool code on-demand
  • API Gateway: Exposes tools via HTTPS
  • IAM Roles: Secure access to AWS services

Cost: 1 million Lambda requests/month = Always Free!

Your First AWS Tool: List EC2 Instances

@tool
def list_ec2_instances(region: str) -> str:
    ec2 = boto3.client("ec2", region_name=region)
    instances = ec2.describe_instances()
    return format_instance_list(instances)

Add IAM permissions → Deploy → Your AI agent can now list your VMs!

Production Best Practices

  • Never use root user for daily work - Create IAM users
  • Enable MFA - Protect your account
  • Use least-privilege IAM policies - Grant minimum permissions
  • Tag everything - Track costs and resources
  • Set billing alerts - Avoid surprise charges

Common Issues & Solutions

  • Can't SSH: Check security group, verify key permissions (chmod 400)
  • Docker permission denied: Add user to docker group, log out/in
  • Container not accessible: Check security group ports
  • MCP server errors: Verify IAM permissions, check Lambda logs

CloudWatch Logs are your best friend for debugging!
